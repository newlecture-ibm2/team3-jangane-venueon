# [통합 기획안] 강의 기반 스터디 연결 및 이벤트 관리 플랫폼: EventBridge (v2.0)

## 1. 프로젝트 개요

### 1.1 비전 및 정의
**"수업 생성부터 목록 탐색, 그리고 후속 스터디 게시판까지 하나로"**
본 플랫폼은 강의/세미나의 기획, 참여 신청, 그리고 종료 후 지속적인 정보 공유를 위한 커뮤니티(스터디) 활동을 지원하는 **웹 기반 통합 이벤트 관리 시스템**입니다. 초기 단계에서는 핵심적인 정보 관리(CRUD)에 집중하여 안정적인 플랫폼 기반을 구축합니다.

### 1.2 핵심 목표
- **기본 기능 충실**: 사용자가 강의를 등록하고, 다른 사용자가 이를 조회하여 신청하는 핵심 프로세스 완성.
- **학습 연속성**: 강의가 끝난 후에도 같은 수강생들이 게시판을 통해 질문과 자료를 공유할 수 있는 환경 제공.
- **확장 가능성**: 향후 고도화된 기술(실시간 중계, 결제 등)을 도입할 수 있도록 유연한 데이터 구조 설계.

## 2. 핵심 CRUD 기능 명세 (Main)

### 2.1 회원 및 프로필 관리 (User)
- **회원가입/로그인**: 이메일 기반의 가입 및 세션/JWT 방식의 로그인.
- **프로필 관리**: 이름, 소개, 사진, 관심 카테고리 설정.
- **역할 구분**: 주최자(Host)와 참여자(Attendee)로 구분하여 접근 권한 제어.

### 2.2 강의 및 이벤트 관리 (Event)
- **강의 등록 (Create)**: 제목, 상세 설명, 일시, 장소, 정원 정보를 입력하여 강의 개설.
- **강의 목록 및 상세 조회 (Read)**: 최신순/인기순 목록 필터링 및 강의 상세 페이지 조회.
- **강의 정보 수정 (Update)**: 본인이 개설한 강의의 세부 사항 수정.
- **강의 삭제 (Delete)**: 이벤트 취소 또는 종료 시 목록에서 삭제/비활성화.

### 2.3 참여 신청 및 티켓 관리 (Registration)
- **참여 신청**: 강의 상세 페이지에서 '신청하기' 버튼을 통해 수강 등록.
- **내 티켓 조회**: 마이페이지에서 본인이 신청한 강의 목록 및 상태(대기/승인) 확인.
- **신청 취소**: 수강 전 신청 내역 취소 처리.

### 2.4 커뮤니티 게시판 관리 (Community)
- **스터디 게시글 작성 (Create)**: 강의 종료 후 생성되는 게시판에 질문, 정리, 토론 글 등록.
- **게시글 조회 (Read)**: 스터디 멤버들이 소통할 수 있는 타임라인 형태의 피드 조회.
- **댓글 작성**: 게시글에 대한 실시간 의견 교환 및 답변 등록.
- **상태 관리**: 작성한 글 및 댓글의 수정/삭제 기능.

## 3. 서비스 아키텍처 (Standard)

### 3.1 시스템 구조
- **Client**: React / Next.js 기반의 웹 프론트엔드.
- **Server**: Spring Boot 기반의 Monolithic API 서버 (초기 개발 편의성 및 CRUD 집중).
- **Database**: PostgreSQL (RDBMS)를 활용한 관계형 데이터 관리.

### 3.2 핵심 데이터 모델 (ERD)
1. **User**: 이메일, 비밀번호, 닉네임, 프로필 정보.
2. **Event**: 주최자 ID, 제목, 내용, 시간, 장소, 정원, 현재 신청자 수.
3. **Registration**: 사용자 ID, 이벤트 ID, 신청 상태, 신청 일시.
4. **Post**: 커뮤니티(이벤트) ID, 작성자 ID, 제목, 내용, 게시글 유형.
5. **Comment**: 게시글 ID, 작성자 ID, 댓글 내용.

## 4. 차후 구현 및 고도화 기술 (Future Scope)

> 초기 CRUD 기반 시스템이 안정화된 후, 경쟁력 강화를 위해 순차적으로 도입할 기술 목록입니다.

### 4.1 실시간 인터랙션 고도화
- **WebSocket (Socket.IO/STOMP)**: 라이브 강의 중 실시간 채팅, 투표, Q&A 기능 도입.
- **인앱 알림**: 새 댓글이나 강의 시간 리마인드를 위한 실시간 푸시 알림.

### 4.2 외부 API 연동 및 전문 기능
- **영상 중계 (Zoom API/WebRTC)**: 외부 툴 이동 없는 플랫폼 내 라이브 강의 송출 및 시청.
- **결제 연동 (PortOne)**: 유료 강의를 위한 신용카드 및 간편 결제 시스템 구축.
- **QR 체크인**: 오프라인 현장 출석 확인을 위한 QR 생성 및 스캐너 모드 개발.

### 4.3 인프라 확장 및 지능화
- **MSA (Microservices Architecture)**: 트래픽 증가 시 서비스별 독립 스케일링을 위해 서버 분산.
- **Kafka / Redis**: 서비스 간 비동기 통신 및 대규모 트래픽 대응을 위한 캐싱 처리.
- **AI 추천 엔진**: 사용자 관심사 기반 맞춤형 강의 추천 시스템.
- **VOD 아카이브**: 라이브 종료 후 녹화 영상 자동 저장 및 다시보기 서비스.

## 5. 개발 로드맵 (단계별)

1. **Phase 1 (CRUD Core)**: 회원, 강의, 신청, 게시글 기본 CRUD 완성 및 MVP 배포.
2. **Phase 2 (UX/UI Polishing)**: 디자인 시스템 적용, 검색 엔진 고도화, 모바일 반응형 최적화.
3. **Phase 3 (Expansion)**: 정산/결제 시스템 및 실시간 채팅 기능 추가.
4. **Phase 4 (Scaling)**: 아키텍처 분리(MSA) 및 AI 추천 시스템 도입.
