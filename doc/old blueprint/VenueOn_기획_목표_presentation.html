Section 01

핵심 컨셉

단순 강의 플랫폼이 아닌, 강의 → 커뮤니티(스터디) 연결이 핵심 가치

→
🎓 스터디 그룹 형성
→
🔄 지속적 학습

📚강의 플랫폼

유·무료 강의를 에디터 기반으로 직접 등록하고, 티켓팅·결제까지 원스톱 제공

🔗강의-스터디 연결

강의 종료 후 참여자들이 스터디룸을 생성하여 지속적인 학습 커뮤니티 형성

🏅뱃지 기반 매칭

수강 완료 뱃지를 통해 유사 관심사 사용자를 연결, 커뮤니티 자생력 강화

🎯 양면 플랫폼 — 강사(HOST)와 학생(USER) 양측 모두에게 가치 제공

Section 02

사용자 역할 구분

  • Zoom 연동 실시간 강의 진행
  • 스터디 그룹 활성화 · 오거나이저 대시보드

  • 마이페이지 → 강의 입장 · 환불/취소

Section 03

강사 (HOST) Flow

기업 서비스 로그인 → 강의 생성 → 진행 → 사후 스터디 관리

🏢 기업 서비스 진입
↓
🔐 강사 로그인 / 회원가입
↓
📝 강의 생성하기
↓
📋 강의 정보 입력
↓
🔧 강의 관리 (대시보드)
↓
📊 참가자 관리 · 모니터링
↓
🎥 Zoom 세션 시작 → 강의 진행
↓
📖 스터디 그룹 활성화

📋 강의 정보 입력 상세

  • 기본 정보 입력 (강의명, 카테고리, 설명)
  • 세션/일정 설정 (날짜, 시간)
  • 티켓 설정 (무료/유료, 가격, 수량)
  • (선택) 온라인 링크 추가 (Zoom)
  • 게시 (Publish) — Draft → Published

🔧 강의 관리 기능

  • 수정 · 상태 변경 (Draft / Published)
  • 링크 추가 · 수강 신청자 승인/거절
  • 대기순번 관리 · 실시간 모니터링

📖 사후 관리

  • Enable Study Group Creation
  • Monitor Study Groups
  • 오거나이저 대시보드

Section 08

뱃지 시스템

수강 이력의 시각화 — 커뮤니티 자생력 강화

🎓 강의 완료
→
🏅 뱃지 발급
→
👤 프로필 노출
→
🔍 뱃지 기반 검색
→
👥 커뮤니티 형성

🏅학생 뱃지

  • 이벤트(강의) 완료 시 뱃지 자동 발급
  • 프로필에 노출, 다른 사용자가 뱃지로 검색
  • 뱃지 미노출 / 관심 뱃지 등록 가능
  • 뱃지 소유자·관심 등록자끼리 커뮤니티 형성

🏢강사(기획자) 뱃지

  • 주최자 뱃지 누적 → 등급 역할 가능
  • 강사 이력·논문 등 포트폴리오 표시
  • 학생이 강사를 구독(팔로우)
  • 신뢰도 지표로 활용

💡 뱃지 기반 매칭 — 핵심 가치

이벤트 기획자가 아닌 일반 사용자들도, 이벤트가 종료된 이후에도 뱃지를 통해
관심사가 같은 사람들끼리 능동적으로 커뮤니티를 형성할 수 있는 자생적 생태계

Section 09

4대 차별화 포인트

🔄 대기열 자동화 시스템

수강 취소 시 다음 대기자에게 자동 결제 기한 부여. 미응답 시 순차적으로 다음 사람에게 넘어가는 정교한 로직. 알림톡 + 응답 시간 제한 + 결제 기한 처리.

🔗 강의-스터디 수직 연결

단순 수강으로 끝나지 않음. 강의 종료 후 관련 스터디룸 생성/참여 유도. 강사·학생 간 지속적 학습 생태계 구축.

🏅 뱃지 기반 매칭

수강 이력이 뱃지로 시각화. 뱃지 기반으로 유사 관심사 사용자 연결. 커뮤니티 자생력 강화. 기획자도 주최자 뱃지 등급 가능.

⭐ 강사 구독 시스템

학생이 강사를 구독(팔로우). 강사 이력, 논문 등 포트폴리오 확인 가능. 검증된 강사 기반 신뢰 플랫폼.

핵심 메시지: 수강 → 커뮤니티 → 지속적 학습으로 이어지는 수직 계열화가 VenueOn의 가장 큰 차별점
