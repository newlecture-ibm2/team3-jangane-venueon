> **Jira**: VO-820 (EC-020 이벤트 기간/상태 관리)
> **작성일**: 2026-04-09
> **브랜치**: `young/feat/VO-820/EC-020-event-dates`

---

## 1. 개요

이벤트의 상태를 **진행 상태(EventStatus)**와 **모집 상태(RecruitmentStatus)** 두 축으로 분리하여 관리한다.

> [!IMPORTANT]
> **핵심 원칙: 세션이 유일한 Source of Truth**
>
> 모든 기간(진행/모집)과 상태는 **세션(Session) 단위**에서만 설정·관리한다.
> 이벤트 레벨의 기간과 상태는 소속 세션들의 값을 **종합(aggregate)**하여 도출한다.
>
> - 이벤트 시작일 = `min(세션들의 startTime)`
> - 이벤트 종료일 = `max(세션들의 endTime)`
> - 이벤트 진행 상태 = 세션들의 진행 상태를 OR 종합
> - 이벤트 모집 상태 = 세션들의 모집 상태를 OR 종합
> - 이벤트 모집 시작일 = `min(세션들의 recruitStartDate)`
> - 이벤트 모집 마감일 = `max(세션들의 recruitEndDate)`

> [!NOTE]
> **모든 날짜 필드는 선택사항(nullable)이다.**
> 호스트가 날짜를 설정하지 않으면 자동 상태 전이가 발생하지 않으며,
> 호스트가 직접 수동으로 상태를 관리한다.

상태 전이는 스케줄러 없이 **조회 시점에 날짜 기반으로 계산**하는 Computed 방식을 채택한다.

## 2. 상태 정의

### 2-1. EventStatus (이벤트 진행 상태)

> DB에는 이벤트 레벨의 수동 상태(DRAFT, PUBLISHED, CANCELLED)만 저장.
> ONGOING, ENDED는 세션의 기간/상태를 종합하여 조회 시 계산.

| 값 | 한글 | 설명 | 전이 방식 |
|---|---|---|---|
| `DRAFT` | 임시저장 | 호스트가 작성 중, 비공개 | 생성 시 "임시저장" 버튼 |
| `PUBLISHED` | 공개(준비중) | 공개됨, 아직 시작 전 | 생성/수정 시 "게시" 버튼 |
| `ONGOING` | 진행중 | 이벤트가 현재 진행 중 | 세션 중 1개라도 진행중이면 자동 |
| `ENDED` | 종료 | 이벤트 종료 | 모든 세션이 종료되면 자동 or 호스트 수동 |
| `CANCELLED` | 취소 | 호스트가 취소 | 호스트 수동 |

#### 상태 전이 다이어그램

```
                  publish()            세션 startTime 도래        모든 세션 endTime 경과
  DRAFT ─────────────▶ PUBLISHED ───────────────▶ ONGOING ───────────────▶ ENDED
    ▲                      │  │                     │  │
    │ save as draft        │  │ (수동 start도 가능)  │  │ (수동 end도 가능)
    │                      │  └─────────────────────┘  └─────────────────────┐
    │                      │ cancel()                   cancel()             │
    │                      ▼                            ▼                    │
    │                  CANCELLED ◀──────────────────────┘                    │
```

#### 이벤트 상태 계산 로직 (세션 종합)

```
1. DB 값이 DRAFT, CANCELLED → 그대로 반환 (수동 확정 상태)

2. DB 값이 PUBLISHED인 경우 → 세션들의 상태를 OR 종합:
   - 1개라도 ONGOING인 세션 있음 → ONGOING
   - ONGOING 없고, 1개라도 시작 전 세션 있음 → PUBLISHED
   - 모든 세션이 ENDED → ENDED

3. DB 값이 ENDED → 그대로 반환 (호스트가 수동 종료한 경우)
```

### 2-2. RecruitmentStatus (모집 상태)

> DB에 저장하지 않음. 세션 단위에서 조회 시 계산 → 이벤트 레벨로 OR 종합.

| 값 | 한글 | 설명 | 판단 기준 |
|---|---|---|---|
| `PENDING` | 모집대기 | 모집 시작 전 | `recruitStartDate` 설정됨 + `now < recruitStartDate` |
| `OPEN` | 모집중 | 참가 신청 가능 | 모집 기간 내 + 잔여석 있음 + 수동 마감 아님 |
| `CLOSED` | 모집마감 | 신청 불가 | 기한 초과 or 정원 초과 or 호스트 수동 마감 |

#### 이벤트 모집 상태 = 세션 OR 종합

```
- 1개라도 OPEN인 세션 있음 → 이벤트 전체 = OPEN
- 모든 세션이 PENDING → 이벤트 전체 = PENDING
- 모든 세션이 CLOSED → 이벤트 전체 = CLOSED
```

## 3. 날짜 필드 설계

### 3-1. Event 레벨 — 날짜 필드 없음 (세션에서 도출)

| 표시 항목 | 도출 방식 |
|---|---|
| 이벤트 시작일 | `min(세션들의 startTime)` — null인 세션 제외 |
| 이벤트 종료일 | `max(세션들의 endTime)` — null인 세션 제외 |
| 이벤트 모집 시작일 | `min(세션들의 recruitStartDate)` — null인 세션 제외 |
| 이벤트 모집 마감일 | `max(세션들의 recruitEndDate)` — null인 세션 제외 |

> [!IMPORTANT]
> `events` 테이블에서 `start_date`, `end_date` 컬럼을 **제거**한다.
> 이벤트의 기간은 항상 세션에서 계산하여 API 응답에 포함한다.

### 3-2. Session 레벨 — 모든 기간의 Source of Truth

| 필드 | 타입 | 필수 | 설명 | 변경 |
|---|---|---|---|---|
| `startTime` | `LocalDateTime` | ❌ | 세션 시작 시각 | nullable로 변경 |
| `endTime` | `LocalDateTime` | ❌ | 세션 종료 시각 | nullable로 변경 |
| `recruitStartDate` | `LocalDateTime` | ❌ | 모집 시작일 | **신규** |
| `recruitEndDate` | `LocalDateTime` | ❌ | 모집 마감일 | **신규** |
| `isRecruitmentClosed` | `boolean` | ✅ | 호스트 수동 마감 플래그 | **신규** |

> [!NOTE]
> **날짜 미설정 시 동작:**
> - `startTime` 없음 → 세션이 자동으로 ONGOING이 되지 않음. 호스트 수동 관리
> - `endTime` 없음 → 세션이 자동으로 ENDED가 되지 않음. 호스트 수동 관리
> - `recruitStartDate` 없음 → 이벤트가 PUBLISHED 되는 즉시 모집 OPEN
> - `recruitEndDate` 없음 → 호스트 수동 마감 또는 정원 초과로만 CLOSED

### 3-3. 타임라인 시나리오

#### 시나리오 A: 날짜 모두 설정 (자동 전이)

```
세션 A: recruitStart=4/1, recruitEnd=4/19, startTime=4/20, endTime=4/20
세션 B: recruitStart=4/1, recruitEnd=4/20, startTime=4/21, endTime=4/21
세션 C: recruitStart=4/1, recruitEnd=4/22, startTime=4/22, endTime=4/22

이벤트 표시:
  시작일 = 4/20 (min startTime)
  종료일 = 4/22 (max endTime)
  모집 시작 = 4/1 (min recruitStartDate)
  모집 마감 = 4/22 (max recruitEndDate)

4/15 → 이벤트: PUBLISHED + 모집 OPEN (전 세션 모집중)
4/20 → 이벤트: ONGOING (A 진행중) + 모집 OPEN (B,C 모집중)
4/21 → 이벤트: ONGOING (B 진행중) + 모집 OPEN (C 모집중)
4/23 → 이벤트: ENDED (전부 종료) + 모집 CLOSED
```

#### 시나리오 B: 날짜 미설정 (수동 관리)

```
세션 A: startTime=null, endTime=null, recruitStartDate=null, recruitEndDate=null

이벤트 게시 → PUBLISHED + 모집 OPEN (날짜 없으므로 즉시)
호스트가 수동 시작 → ONGOING
호스트가 수동 마감 → 모집 CLOSED
호스트가 수동 종료 → ENDED
```

#### 시나리오 C: 부분 설정 (하이브리드)

```
세션 A: startTime=4/20, endTime=null, recruitEnd=4/19

→ 4/19에 모집 자동 마감
→ 4/20에 자동으로 ONGOING
→ 종료는 호스트가 수동으로 결정 (유동적 운영)
```

## 4. 도메인 모델 변경

### 4-1. Enum

```java
// EventStatus.java — PREPARING 제거
public enum EventStatus {
    DRAFT, PUBLISHED, ONGOING, ENDED, CANCELLED
}

// RecruitmentStatus.java — 신규 생성
public enum RecruitmentStatus {
    PENDING, OPEN, CLOSED
}
```

### 4-2. Event 도메인 모델

```java
public class Event {
    // startDate, endDate 필드 제거
    // status는 DB 저장용 (DRAFT, PUBLISHED, CANCELLED만 직접 저장)

    /**
     * 세션 목록을 받아 이벤트의 실제 진행 상태를 계산.
     * DB에 저장된 status 값 + 세션들의 기간을 종합하여 보정한다.
     */
    public EventStatus getEffectiveStatus(List<EventSession> sessions) {
        // 수동 확정 상태는 보정 불필요
        if (status == EventStatus.DRAFT || status == EventStatus.CANCELLED) {
            return status;
        }

        // 호스트가 수동으로 종료한 경우
        if (status == EventStatus.ENDED) {
            return EventStatus.ENDED;
        }

        // PUBLISHED 상태 → 세션 종합으로 보정
        if (sessions == null || sessions.isEmpty()) {
            return status;
        }

        LocalDateTime now = LocalDateTime.now();
        boolean hasOngoing = false;
        boolean hasNotEnded = false;

        for (EventSession session : sessions) {
            EventStatus sessionStatus = session.getSessionStatus();
            if (sessionStatus == EventStatus.ONGOING) hasOngoing = true;
            if (sessionStatus != EventStatus.ENDED) hasNotEnded = true;
        }

        if (hasOngoing) return EventStatus.ONGOING;
        if (!hasNotEnded) return EventStatus.ENDED;   // 모든 세션 종료
        return EventStatus.PUBLISHED;                  // 아직 시작 전 세션 있음
    }

    /**
     * 세션 목록에서 이벤트 표시용 시작일 도출
     */
    public static LocalDateTime deriveStartDate(List<EventSession> sessions) {
        return sessions.stream()
                .map(EventSession::getStartTime)
                .filter(Objects::nonNull)
                .min(LocalDateTime::compareTo)
                .orElse(null);
    }

    /**
     * 세션 목록에서 이벤트 표시용 종료일 도출
     */
    public static LocalDateTime deriveEndDate(List<EventSession> sessions) {
        return sessions.stream()
                .map(EventSession::getEndTime)
                .filter(Objects::nonNull)
                .max(LocalDateTime::compareTo)
                .orElse(null);
    }

    /**
     * 세션 목록을 OR 종합하여 이벤트 모집 상태 도출
     */
    public static RecruitmentStatus aggregateRecruitmentStatus(List<EventSession> sessions) {
        if (sessions == null || sessions.isEmpty()) {
            return RecruitmentStatus.CLOSED;
        }

        boolean hasOpen = false;
        boolean hasPending = false;

        for (EventSession session : sessions) {
            RecruitmentStatus rs = session.getRecruitmentStatus();
            if (rs == RecruitmentStatus.OPEN) hasOpen = true;
            if (rs == RecruitmentStatus.PENDING) hasPending = true;
        }

        if (hasOpen) return RecruitmentStatus.OPEN;
        if (hasPending) return RecruitmentStatus.PENDING;
        return RecruitmentStatus.CLOSED;
    }

    /**
     * 세션 목록에서 이벤트 표시용 모집 시작일 도출
     */
    public static LocalDateTime deriveRecruitStartDate(List<EventSession> sessions) {
        return sessions.stream()
                .map(EventSession::getRecruitStartDate)
                .filter(Objects::nonNull)
                .min(LocalDateTime::compareTo)
                .orElse(null);
    }

    /**
     * 세션 목록에서 이벤트 표시용 모집 마감일 도출
     */
    public static LocalDateTime deriveRecruitEndDate(List<EventSession> sessions) {
        return sessions.stream()
                .map(EventSession::getRecruitEndDate)
                .filter(Objects::nonNull)
                .max(LocalDateTime::compareTo)
                .orElse(null);
    }

    // --- 수동 상태 전이 메서드 ---

    public void publish() {
        if (this.status != EventStatus.DRAFT) {
            throw new IllegalStateException("DRAFT 상태에서만 공개 가능합니다.");
        }
        this.status = EventStatus.PUBLISHED;
    }

    public void end() {
        this.status = EventStatus.ENDED;
    }

    public void cancel() {
        this.status = EventStatus.CANCELLED;
    }
}
```

### 4-3. EventSession 도메인 모델

```java
public class EventSession {
    // 기존 필드 유지 (startTime, endTime은 nullable)
    // + 추가:
    private LocalDateTime recruitStartDate;    // nullable
    private LocalDateTime recruitEndDate;      // nullable
    private boolean isRecruitmentClosed;       // 호스트 수동 마감 플래그

    /**
     * 세션의 진행 상태를 계산.
     * 날짜가 설정되어 있으면 자동 계산, 없으면 수동 관리 필요.
     */
    public EventStatus getSessionStatus() {
        LocalDateTime now = LocalDateTime.now();

        // endTime이 설정되어 있고 경과했으면 → ENDED
        if (endTime != null && !now.isBefore(endTime)) {
            return EventStatus.ENDED;
        }

        // startTime이 설정되어 있고 도래했으면 → ONGOING
        if (startTime != null && !now.isBefore(startTime)) {
            return EventStatus.ONGOING;
        }

        // 날짜 미설정이거나 아직 시작 전 → PUBLISHED (대기)
        return EventStatus.PUBLISHED;
    }

    /**
     * 세션의 모집 상태를 계산.
     * 우선순위: 호스트 수동 마감 > 정원 초과 > 날짜 기반 > 기본값(OPEN)
     */
    public RecruitmentStatus getRecruitmentStatus() {
        // 1. 호스트가 수동으로 마감한 경우 → 무조건 CLOSED
        if (isRecruitmentClosed) {
            return RecruitmentStatus.CLOSED;
        }

        // 2. 정원 초과 (maxAttendees=0은 무제한)
        if (maxAttendees > 0 && currentAttendees >= maxAttendees) {
            return RecruitmentStatus.CLOSED;
        }

        LocalDateTime now = LocalDateTime.now();

        // 3. 모집 시작일이 설정되어 있고 아직 안 됐으면 → PENDING
        if (recruitStartDate != null && now.isBefore(recruitStartDate)) {
            return RecruitmentStatus.PENDING;
        }

        // 4. 모집 마감일이 설정되어 있고 지났으면 → CLOSED
        if (recruitEndDate != null && now.isAfter(recruitEndDate)) {
            return RecruitmentStatus.CLOSED;
        }

        // 5. 위 조건에 해당 안 하면 → OPEN
        return RecruitmentStatus.OPEN;
    }
}
```

## 7. 호스트 상태 관리 API

### 7-1. 이벤트 진행 상태 수동 변경

```
PATCH /host/events/{id}/status?status=ENDED        ← 수동 종료
PATCH /host/events/{id}/status?status=CANCELLED    ← 취소
PATCH /host/events/{id}/status?status=PUBLISHED    ← 게시 (기존)
```

### 7-2. 모집 수동 마감/재개

세션 단위 마감:
```
PATCH /host/events/{eventId}/sessions/{sessionId}/recruitment
Body: { "closed": true }
```

이벤트 전체 세션 일괄 마감:
```
PATCH /host/events/{eventId}/recruitment
Body: { "closed": true }
→ 해당 이벤트의 모든 세션 isRecruitmentClosed = true
```

- `closed: true` → 즉시 모집 마감
- `closed: false` → 마감 해제 (기한/정원이 허용하면 다시 OPEN)

public Order createOrder(Long sessionId, Long userId, int quantity) {
    EventSession session = sessionRepository.findById(sessionId)...;

    // 모집 상태 검증
    if (session.getRecruitmentStatus() != RecruitmentStatus.OPEN) {

            "잔여석이 부족합니다.");
    }

}
