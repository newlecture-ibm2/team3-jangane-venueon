# VenueOn MVP v3 발표 대본

> 슬라이드별 간략 발표 스크립트  
> 예상 소요시간: 약 10~12분

---

## 슬라이드 0 — Cover

> 안녕하세요, 저희 팀이 기획한 **VenueOn** 프로젝트를 소개하겠습니다.  
> VenueOn은 **유료·무료 이벤트 중계와 커뮤니티를 결합한 플랫폼**입니다.  
> 기술 스택은 Spring Boot, Next.js 14, Docker 기반이며,  
> MSA 3개 서비스, Entity 11개, API 42개, 페이지 8개 규모의 MVP입니다.

---

## 슬라이드 1 — MVP 기능 범위

> MVP에서 제공하는 **7가지 핵심 기능**입니다.  
> 
> 첫째, **JWT 기반 회원가입/로그인**으로 기업·일반 사용자를 구분합니다.  
> 둘째, **이벤트 CRUD** — 에디터(폼)를 통해 직접 이벤트를 등록합니다.  
> 셋째, **마이페이지**에서 내 이벤트, 구매내역, 참여 이력을 관리할 수 있고,  
> 넷째, **티켓팅과 포트원 결제 연동**으로 유료·무료 티켓 구매가 가능합니다.  
> 
> 또한 **이벤트 기반 커뮤니티**, **커뮤니티 게시글/후기** 기능을 제공하며,  
> 마지막으로 **기업/기관 전용 페이지**에서 프로필과 주최 이벤트를 노출합니다.

---

## 슬라이드 2 — 타겟 사용자 & 정책

> 타겟 사용자는 크게 **두 분류**입니다.  
> 
> **기획자(HOST)**는 기업, 공공기관, 사업자로서 이벤트를 생성·관리·판매합니다.  
> 사업자등록번호 또는 기관 인증으로 가입하며, 기업 전용 프로필 페이지를 보유합니다.  
> 
> **일반 사용자(USER)**는 개인 사용자로, 이벤트를 탐색하고 티켓을 구매합니다.  
> 이메일 기반으로 가입하고, 구매내역·참여 이력을 관리합니다.  
> 
> 정책적으로, 이벤트 생성은 **HOST만 가능**하고, 모든 콘텐츠는 본인 작성분만 수정·삭제할 수 있습니다. 향후 ADMIN 역할 추가도 가능한 구조입니다.

---

## 슬라이드 3 — 이벤트 등록 방식

> 이벤트 등록은 **에디터(폼) 기반 5단계 프로세스**입니다.  
> 
> **Step 1~2**에서 이벤트명, 카테고리, 유형, 썸네일, 일정, 장소, 최대 참석자 수를 입력하고,  
> **Step 3~4**에서 세션(발표자·시간)을 복수 등록하고, 티켓 타입(무료/일반/VIP/얼리버드)과 가격·수량을 설정합니다.  
> **Step 5**에서 리치 텍스트 에디터로 상세 설명을 작성한 뒤,  
> 미리보기를 확인하고 **임시저장(Draft) 또는 발행(Published)**을 선택합니다.  
> 
> 폼에 데이터를 입력하면 **상세 페이지, 캘린더, 일정 정보가 자동 구성**됩니다.

---

## 슬라이드 4 — MSA 아키텍처

> 백엔드는 **3개 마이크로서비스**로 분리했습니다.  
> 
> 프론트엔드로부터 HTTP 요청이 들어오면 **Spring Cloud Gateway**가 JWT 검증과 라우팅을 수행합니다.  
> 
> **User Service(8081)**는 회원가입, 로그인, JWT, 프로필, 마이페이지를 담당하며, user_db와 Redis를 사용합니다.  
> **Event Service(8082)**는 이벤트 CRUD, 세션, 티켓, 주문/결제를 담당하며, event_db와 포트원을 연동합니다.  
> **Community Service(8083)**는 커뮤니티 CRUD, 게시글, 댓글, 후기를 담당하며, community_db를 사용합니다.  
> 
> 서비스 간 통신은 **OpenFeign(REST)**으로 처리합니다.

---

## 슬라이드 5 — ERD (11개 Entity)

> ERD는 총 **11개 Entity**로 구성했습니다.  
> 
> 핵심 관계를 보면, **USER**가 **EVENT**를 생성하고, **ORDER**를 통해 구매합니다.  
> EVENT 아래에 **SESSION**, **TICKET**, **COMMUNITY**가 연결되고,  
> TICKET은 ORDER와 연결됩니다.  
> COMMUNITY 아래에 **POST**, **COMMENT**가 계층 구조를 이룹니다.  
> 
> 이 구조로 요구사항인 **최소 Entity 5개, API 20개, 서비스 3개를 모두 충족**합니다.

---

## 슬라이드 6 — 기술 스택

> 기술 스택입니다.  
> 
> **프론트엔드**는 Next.js 14 App Router로 SSR/SSG를 활용하고, CSS Module로 컴포넌트별 스코프를 관리합니다.  
> **백엔드**는 Spring Boot 3, Java 17 기반으로 헥사고날 아키텍처를 적용했고, Spring Cloud Gateway와 OpenFeign을 사용합니다.  
> **데이터**는 PostgreSQL 15를 서비스별 독립 DB 3개로 운영하고, Redis 7으로 JWT 블랙리스트와 캐시를 처리합니다.  
> **인프라**는 Docker Compose로 구성하고, GitHub Actions로 CI/CD를 자동화합니다.

---

## 슬라이드 7 — 헥사고날 아키텍처

> 백엔드 설계의 핵심인 **헥사고날 아키텍처(Ports & Adapters)**입니다.  
> 
> 기존 레이어드 아키텍처는 Controller→Service→Repository로 계층 간 강결합이 발생하여, 기능 추가 시 전 계층을 수정해야 합니다.  
> 
> 저희가 적용한 **헥사고날 구조**에서는  
> Web Controller(Adapter In)이 Port In을 통해 UseCase를 호출하고,  
> UseCase가 Port Out을 통해 JPA Repository(Adapter Out)에 접근합니다.  
> 
> **도메인이 외부에 의존하지 않아서**, 새로운 기능은 UseCase만 추가하면 됩니다. 이것이 바로 **개방-폐쇄 원칙(OCP)**을 지키는 설계입니다.

---

## 슬라이드 8 — API 목록 (42개)

> 총 **42개 API** 목록입니다.  
> 
> **User Service 7개**: 회원가입, 로그인, 토큰 갱신, 로그아웃, 프로필 조회·수정, 사용자 조회  
> **Event Service 18개**: 이벤트 CRUD 7개, 세션 4개, 티켓 2개, 주문 4개, 결제 웹훅 1개  
> **Community Service 17개**: 커뮤니티 CRUD·가입·탈퇴·멤버 9개, 게시글 5개, 댓글 3개  
> 
> 요구사항 최소 20개를 **두 배 이상** 충족하는 규모입니다.

---

## 슬라이드 9 — 페이지 구성 & 기업페이지 흐름

> 총 **8개 페이지**로 구성했습니다.  
> 
> 메인 홈(공동 랜딩), 로그인/가입, 이벤트 상세, 이벤트 생성,  
> 커뮤니티, 마이페이지, 기업/기관 페이지, 기업 대시보드입니다.  
> 
> 기업 페이지 접근 흐름은 심플합니다.  
> **공동 랜딩**에서 헤더의 "기업페이지" 버튼을 클릭하면  
> 해당 기업/기관의 **소개, 주최 이벤트, 연락처**가 표시되는 전용 페이지로 이동합니다.

---

## 슬라이드 10 — 프로젝트 일정

> 프로젝트 일정은 **MVP 1주 완성** 계획입니다.  
> 
> **1~2일차**: 요구사항/WBS/ERD/API 설계 및 Docker, 헥사고날 아키텍처 구축  
> **3~4일차**: 인증(User), 이벤트(Event), 커뮤니티(Community) 핵심 백엔드 API 개발  
> **5~6일차**: Next.js UI 구현, API 연동, 포트원 결제 연동 테스트  
> **7일차**: 통합 테스트, 버그 수정, 최종 시연 준비  
> 
> **3월 30일(월) 시작, 4월 5일(일) 최종 시연·데모**입니다.

## 슬라이드 11 — Summary

> 마지막으로 정리하겠습니다.  
> 
> **타겟**은 기업/공공기관(기획자)과 일반 사용자(참여자)이며,  
> **등록 방식**은 에디터(폼) 기반 Step-by-Step 이벤트 직접 등록입니다.  
> **페이지 흐름**은 공동 랜딩에서 헤더 버튼으로 기업 전용 페이지에 진입합니다.

> **아키텍처**는 MSA 3개 서비스에 헥사고날 구조를 적용했고,  
> **기술 스택**은 Spring Boot 3 + Next.js 14 + PostgreSQL×3 + Redis + Docker Compose입니다.  
> 
> MSA 3개, Entity 11개, API 42개, 페이지 8개, **7일 완성** 계획입니다.
