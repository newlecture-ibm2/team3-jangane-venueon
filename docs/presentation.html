TECHNICAL ARCHITECTURE
04

Domain 01: Core Economy

Event, Session, Ticket & Payment

계층형 상품 설계

Event - Session - Ticket 3단계 구조. 호스트가 자유롭게 세션을 묶어 패키지 티켓을 구성하는 'Smart Default' 전략 적용

재고 정합성 보호

묶음 상품과 개별 상품 간의 복합 재고 관리. 비관적 락(Pessimistic Locking)으로 인기 세션의 초과 판매(Overselling) 원천 차단

트랜잭션 최적화

결제 완료 후 SMTP/Toss 통신 지연을 @Async 비동기 처리로 해결. 구글 캘린더 즉시 연동 및 장바구니 중복 구매 선제 방어 적용

EVENT & PAYMENT SYSTEM
05

Domain 02: Social & Retention

Community, Badge & MyPage

도메인 융합 권한

CommunityPermissionService: 결제 내역(Order)과 뱃지 보유 여부(Badge)를 통합 검증하여 작성 권한을 부여하는 다중 도메인 체크

입체형 리텐션 뱃지

8각형 클립패스 기반의 홀로그램 UI 뱃지. 단순 수집을 넘어 전용 커뮤니티 입장권 역할을 수행하는 소셜 성장 모델 구축

통합 마이페이지

실시간 데이터 기반 이벤트 자동 분류(진행전/중/종료), 상태 머신 기반의 1:1 문의 CS 루프, 전역 프로필 실시간 동기화 구현

COMMUNITY & USER EXPERIENCE
06

Domain 03: Management & Trust

Admin, Host & Security

격리된 워크스페이스

Admin, Host, Manager 역할별 인트라넷 급 독립 패널 구축. 권한 위임을 통해 특정 커뮤니티 전용 부관리자 기능 수행 가능

운영 효율화 루틴

JPA Specification 기반의 동적 필터 쿼리. 관리자 실수 방지를 위한 Step-wise Modal 및 통계 시각화 대시보드 제공

BFF 보안 아키텍처

Next.js BFF 레이어에서 JWT 은닉 관리. XSS 차단 및 OAuth2 / Local 인증 공급자별 조건부 렌더링 최적화

SYSTEM ADMINISTRATION & SECURITY
07

Trouble Shooting

"기술 부채를 해결하는 엔지니어링"

💡 SMTP/Toss Blocking 문제

동기 메일링 지연(15초)을 @Async & 스레드풀 최적화로 해결 (Non-blocking 전환)

💡 GDPR & 재가입 무결성

Soft Delete 한계 극복을 위해 Hard Delete + Cascade 정책 수립 및 무결성 확보

💡 Hydration 엇박자 오류

SSR/CSR간 인증 상태 불일치를 Zustand & checkSession 동기화 패턴으로 영구 해결

💡 도커 볼륨 서빙 최적화

Spring 스레드 점유 없이 Nginx가 파일 직접 서빙하도록 인프라 최적화 (응답속도 0.5ms)
