## 1. 핵심 컨셉

**온라인 강의 수강 → 사후 스터디 그룹 형성 → 지속적 학습까지 수직 계열화된 교육·이벤트 경험 제공**

- 단순 강의 플랫폼이 아닌, **강의 → 커뮤니티(스터디) 연결**을 핵심 가치로 함
- 강의가 끝난 뒤에도 참여자들이 **스터디룸**을 만들어 지속적인 학습 가능
- 강사(기획자)와 학생(참여자) 양측 모두에게 가치를 제공하는 양면 플랫폼

## 2. 사용자 역할 구분

### 2-1. 강사 (Instructor / HOST)

| 단계 | 행동 | 상세 |
|------|------|------|
| **진입** | 기업 서비스 로그인 / 회원가입 | 사업자·기관 인증 기반 가입 |
| **강의 생성** | 강의 정보 입력란 | 기본 정보 입력, 세션/일정 설정, 티켓 설정, (선택) 온라인 링크 추가, 게시(Publish) |
| **강의 관리** | 강사 대시보드 | 수정, 상태 변경 (Draft/Published), 링크 추가 |
| **참가자 관리** | 수강 신청자 승인/거절 | 대기순번 관리, 실시간 모니터링 |
| **강의 진행** | Zoom 세션 연동 | Zoom 세션 연결 → 발행(Publish) → 강의 시작 |
| **사후 관리** | 스터디 그룹 활성화 | Enable Study Group Creation → 모니터링 → 오거나이저 대시보드 |

### 2-2. 학생 (Student / USER)

| 단계 | 행동 | 상세 |
|------|------|------|
| **탐색** | 강의 탐색 / 장바구니·찜 | 관심 강의 장바구니 담기 / 찜하기 |
| **수강 신청** | 결제 진행 | 정원 내 → 결제 완료, 정원 초과 → 대기순번 부여 |
| **대기열** | 자동 승급 시스템 | 취소 발생 시 자동 승급 (상세 아래 참고) |
| **수강** | 마이페이지에서 관리 | 강의 상세 확인, Zoom 입장, 환불/취소 |
| **스터디 연계** | 스터디 추천 섹션 | 강의 완료 후 관련 스터디 참여 또는 신규 생성 |
| **이력 관리** | 강의 뱃지 | 수강 완료 시 뱃지 부여, 학습 이력 관리 |

## 3. 핵심 Flow Chart

### 3-1. 강사 Flow

```
기업 서비스 진입
  ↓
강사 로그인 ─(no)→ 회원 가입
  ↓ (yes)
강의 생성하기
  ↓
강의 정보 입력란
  • 기본 정보 입력
  • 세션/일정 설정
  • 티켓 설정
  • (선택) 온라인 링크 추가
  • + 게시(Publish)
  ↓
강의 관리
  • 수정
  • 상태 변경 (Draft / Published)
  • 링크 추가
  ↓
강사 대시보드
  ↓
Zoom 세션 연결? ─(연결)→ Publish Lecture
  ↓
참가자 관리 / 등록 확인 (View Registrations)
  ↓
강의 상태 모니터링 (Monitor Lecture Status)
  ↓
강의 시간? ─(Not Yet)→ 대기
             ↓ (Ready)
Start Zoom Session → Conduct Lecture → End Lecture
  ↓
Enable Study Group Creation → Monitor Study Groups → Organizer Dashboard
```

### 3-2. 학생 Flow

```
강의 탐색
  ↓
수강 신청 (Registration)
  ↓
결제 성공? ─(no)→ 재시도 / Free
            ↓ (yes)
Confirm Registration
  ↓
Access My Lectures (내 강의실)
  ↓
Wait for Lecture Time → Join Online Lecture via Zoom → Attend Lecture → Complete Lecture
  ↓
Enter Study Recommendation Section (스터디 추천 섹션)
  ↓
View Study Groups Related to Lecture
  ↓
Join or Create? ─(Create New)→ Create New Study Group → Set Study Details, Purpose, Schedule
                  ↓ (Join Existing)
                Select Study Group → Request to Join
```

## 4. 주요 기능 모듈 상세

### 4-1. 강의 생성 (강사)

- **기본 정보 입력**: 강의명, 카테고리, 설명
- **세션/일정 설정**: 날짜, 시간, 복수 세션 등록
- **티켓 설정**: 무료/유료, 가격, 수량, 판매 기간
- **(선택) 온라인 링크 추가**: Zoom 등 화상 회의 링크
- **게시 (Publish)**: Draft → Published 상태 전환

### 4-2. 강사 대시보드

- 강의 상태 관리 (Draft / Published)
- 참가자 리스트 및 결제 상태 조회
- 수강 신청자 승인/거절
- 대기순번 관리
- 실시간 모니터링

### 4-3. 대기열 자동 승급 시스템 (학생)

> **핵심 차별점**: 정원 초과 시 정교한 대기열 관리

- 대기순번 관리
- 취소 시 자동 승급 여부 판단
- **승급 알림** 전송 (알림톡)
- **응답 시간 제한** 설정
- **결제 기한 처리** (기한 내 미결제 시 자동 패스)
- 미응답 시 **다음 사람에게 넘기기**

### 4-4. 학생 마이페이지

| 메뉴 | 기능 | 상세 |
|------|------|------|
| **강의 장바구니/찜하기?** | 관심 강의 관리 | 찜 목록, 장바구니 |
| **예약한 강의** | 대기 상태 확인 | 대기순번, 승급 대기, 결제 기한 |
| **구독한 강사** | 강사 팔로잉 | 강사이력, 강사 논문 등 페이지 |
| **내가 듣는 강의 관리** | 수강 중인 강의 | 강의 상세 → 환불/취소, 입장하기 |
| **내 스터디룸** | 스터디 커뮤니티 | 게시물 생성, 게시물 보기, 댓글 쓰기, 실시간 채팅 |
| **내가 완료한 강의** | 학습 이력 | 강의 뱃지 부여 |

### 4-5. 스터디룸 관리

- **커뮤니티 정보 수정** (이름, 설명, 공개 여부)
- **게시글 목록 보기**
- **게시글 관리** (삭제, 공지 고정)
- **공지글 고정** (pin)
- **신고된 글 처리** (선택)
- **멤버 관리** (리스트, 강퇴)
- **입장 조건 설정** (구매자한)

### 4-6. 강의 뱃지 시스템

- 이벤트(강의) 완료 시 해당 데이터가 **뱃지**로 발급
- 프로필에 노출, 다른 사용자가 뱃지로 검색 가능
- 뱃지 미노출 / 관심 뱃지 등록 설정 가능
- 뱃지 소유자·관심 등록자끼리 **능동적 커뮤니티 형성**
- 기획자(강사)도 **주최자 뱃지** 누적으로 등급 역할 가능

│                       VenueOn 플랫폼                           │
├───────────────────────┬───────────────────────────────────────┤
│   강사 (HOST) 영역     │         학생 (USER) 영역               │

## 6. 기획 핵심 포인트 (차별화 요소)

### ✅ 대기열 자동화 시스템
- 수강 취소 시 다음 대기자에게 자동 결제 기한 부여
- 미응답 시 순차적으로 다음 사람에게 넘어가는 정교한 로직
- 알림톡 + 응답 시간 제한 + 결제 기한 처리

### ✅ 강의-스터디 수직 연결
- 단순 수강으로 끝나지 않음
- 강의 종료 후 관련 스터디룸 생성/참여 유도
- 강사·학생 간 지속적 학습 생태계 구축

### ✅ 뱃지 기반 매칭
- 수강 이력이 뱃지로 시각화
- 뱃지 기반으로 유사 관심사 사용자 연결
- 커뮤니티 자생력 강화

### ✅ 강사 구독 시스템
- 학생이 강사를 구독(팔로우)
- 강사 이력, 논문 등 포트폴리오 확인 가능
