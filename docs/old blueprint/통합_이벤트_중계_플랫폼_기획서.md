> **프로젝트명:** EventBridge — 오프라인 행사를 온라인과 함께, 올인원 이벤트 플랫폼  
> **작성일:** 2026-03-25  
> **기반 문서:** K-DT 선도기업 아카데미 1차_v0.50.pdf, 주제_후보_분석.md, kdt_forum_analysis.md

## 📌 1. 프로젝트 개요

### 한 줄 정의

> **"Cloud Native 기반으로 오프라인 세미나·행사를 온라인과 연결하고, 티켓팅·웨비나·커뮤니티·인터랙티브 기능을 하나의 플랫폼에서 제공하는 통합 이벤트 중계 플랫폼"**

### 핵심 컨셉

| 키워드 | 설명 |
|--------|------|
| **오프라인 ↔ 온라인 브릿지** | 오프라인 세미나/행사를 온라인으로 동시 참여 가능 |
| **올인원 플랫폼** | 티켓팅, 웨비나, 커뮤니티, 투표 등 외부 플랫폼 없이 한 번에 해결 |
| **커뮤니티 형성** | 이벤트 주제 기반 참여자 간 커뮤니케이션 유도 |
| **규모 무관** | 대형 컨퍼런스부터 소규모 원데이 클래스까지 모두 개설 가능 |
| **이벤트 중계** | 세미나, 소규모 클래스를 중계하는 플랫폼 역할 |

## 📌 2. As-Is → To-Be

| 구분 | 내용 |
|------|------|
| **As-Is (현재 문제)** | ① 오프라인 행사 운영 시 티켓팅(인터파크), 웨비나(Zoom), 커뮤니티(카페/밴드), 투표(구글폼) 등 **4~5개 외부 플랫폼** 필요 ② 참여자는 여러 플랫폼을 오가며 **접근성 저하** ③ 소상공인/기업이 자체 이벤트 시스템 구축 불가 ④ 행사 후 참여자 간 소통 단절 → **커뮤니티 형성 실패** ⑤ 이벤트 데이터(참여율, 반응 등) 분산 → **통합 분석 불가** |
| **To-Be (해결)** | ① 이벤트 개설~티켓팅~중계~커뮤니티~사후분석까지 **원스톱 플랫폼** ② 사용자는 한 사이트에서 모든 것을 해결 ③ 기업/소상공인이 **노코드 에디터**로 이벤트 페이지 직접 생성 ④ 주제 기반 커뮤니티 자동 형성 → **지속적 참여 유도** ⑤ 통합 대시보드로 이벤트 성과 한눈에 확인 |

## 📌 3. 타겟 사용자

### 3-1. 이벤트 주최자 (Host)

| 유형 | 예시 | 니즈 |
|------|------|------|
| **기업** | IT 기업 세미나, 신제품 발표회 | 전문적인 이벤트 운영 + 고객 데이터 수집 |
| **소상공인** | 원데이 클래스, 체험 이벤트 | 간편한 이벤트 개설 + 판촉 연동 |
| **교육 기관** | 세미나, 워크숍, 특강 | 온·오프라인 동시 운영 + 출석 관리 |
| **커뮤니티 리더** | 팬미팅, 동호회 모임, 스터디 | 커뮤니티 관리 + 소통 채널 |
| **개인 크리에이터** | 강연, 클래스, 웨비나 | 수익화 + 팬 관리 |

### 3-2. 이벤트 참여자 (Attendee)

| 니즈 | 제공 가치 |
|------|-----------|
| 한 곳에서 모든 이벤트 탐색 | 통합 이벤트 마켓플레이스 |
| 간편한 티켓 구매/관리 | 원클릭 티켓팅 + 모바일 QR 입장 |
| 온라인으로도 현장 체험 | 실시간 웨비나 + 인터랙티브 참여 |
| 같은 관심사 사람들과 소통 | 이벤트 기반 커뮤니티 |
| 관련 이벤트 추천 | AI 기반 맞춤 추천 |

## 📌 7. 사용자 플로우

### 7-1. 주최자(Host) 플로우

```
이벤트 개설 → 상세페이지 에디터로 꾸미기 → 세션/프로그램 등록
    → 티켓 설정 (유료/무료) → 이벤트 공개
    → 참가자 관리 → 웨비나 중계 설정 → 실시간 운영 (투표/Q&A/퀴즈)
    → 이벤트 종료 → 설문 발송 → 성과 분석 대시보드 확인
    → 정산 처리
```

### 7-2. 참여자(Attendee) 플로우

```
이벤트 탐색/검색 → 이벤트 상세 확인 → 티켓 구매/무료 등록
    → 리마인드 알림 수신 → 당일: QR 입장(오프라인) or 웨비나 접속(온라인)
    → 실시간 참여 (투표, Q&A, 채팅, 퀴즈) → 이모지 리액션
    → 이벤트 종료 → 커뮤니티 참여 → 후기 작성 → 설문 응답
    → 아카이브 다시보기 → 추천 이벤트 탐색 → 포인트/레벨업
```

## 📌 8. 차별화 포인트

### 기존 서비스 대비 차별화

| 기존 서비스 | 한계 | EventBridge 차별화 |
|------------|------|-------------------|
| **인터파크/예스24 티켓** | 티켓팅만 가능, 행사 운영 기능 없음 | 티켓팅 + 운영 + 커뮤니티 통합 |
| **Zoom/Teams** | 웨비나만 가능, 커뮤니티/티켓팅 없음 | 웨비나 + 인터랙티브 기능 통합 |
| **네이버 카페/밴드** | 커뮤니티만 가능, 이벤트 운영 기능 없음 | 커뮤니티 + 이벤트 연동 |
| **구글폼** | 설문/투표만, 실시간 아님 | 실시간 투표/Q&A/퀴즈 통합 |
| **Eventbrite** | 해외 서비스, 한국 결제 연동 약함 | 국내 PG 연동 + 카카오 알림 |

### 핵심 차별화 3가지

1. **🔗 올인원**: 주최자는 외부 플랫폼 없이 이벤트의 A-Z를 해결
2. **🎮 인터랙티브**: 실시간 투표·Q&A·퀴즈·이모지로 참여자 몰입 극대화
3. **👥 커뮤니티 연속성**: 이벤트가 끝나도 주제 기반 커뮤니티가 이어짐

## 📌 16. 발표 시 데모 시나리오 (예시)

> 1. **주최자 시나리오**: "소상공인이 원데이 쿠킹 클래스를 개설하는 과정"
>    - 이벤트 생성 → 상세페이지 꾸미기 → 티켓 설정 → 공개
>
> 2. **참여자 시나리오**: "사용자가 이벤트를 발견하고 참여하는 과정"  
>    - 이벤트 탐색 → 티켓 구매 → 당일 QR 입장 → 웨비나 접속 → 실시간 투표 → 채팅
>
> 3. **인터랙티브 시나리오**: "이벤트 중 실시간 기능 시연"
>    - 실시간 투표 (이상형 월드컵) → Q&A → 퀴즈쇼 → 이모지 리액션
>
> 4. **사후 시나리오**: "이벤트 후 커뮤니티 형성"
>    - 커뮤니티 자동 생성 → 후기 작성 → 추천 이벤트 탐색
