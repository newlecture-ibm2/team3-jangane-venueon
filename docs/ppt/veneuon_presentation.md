## 1. 프로젝트 개요 (Overview)
- **명칭**: VenueOn (베뉴온)
- **한 줄 정의**: 누구나 쉽게 이벤트를 열고, 참여하며, 소통할 수 있는 **이벤트 관리 및 커뮤니티 플랫폼**
- **목표**: 
    - 복잡한 티켓팅 및 세션 관리 프로세스 단순화
    - 이벤트 특성에 맞는 리치 컨텐츠 제공
    - 지역 기반의 정교한 이벤트 탐색 기능 제공

---

## 2. VenueOn만의 차별점 (Differentiation)
- **이벤트 전 주기를 아우르는 커뮤니티**: 단순한 예매 대행 플랫폼을 넘어, 이벤트 전/중/후에 걸쳐 참가자와 호스트가 소통할 수 있는 **'밀착형 커뮤니티'** 제공
- **리치 에디터 기반 정밀 컨텐츠**: Tiptap 에디터를 통한 고품질 이벤트 상세 페이지로 호스트의 브랜딩 강화 및 정보 전달력 극대화
- **유연한 세션 대응력**: 온/오프라인이 혼합된 하이브리드 세션을 하나의 이벤트 내에서 통합 관리하는 독보적인 유연성

---

## 2. 핵심 기능 (Key Features)
- **이벤트 생명주기 관리**: 호스트가 이벤트를 기획, 등록, 운영하고 상태를 수동으로 제어
- **온/오프라인 세션 시스템**: 하나의 이벤트 내에서 다양한 시간대와 장소(Zoom 또는 오프라인 장소)를 가진 세션 운영
- **정교한 티켓팅 및 결제**: Toss Payments 연동을 통한 실시간 결제 및 구매자 전용 링크 보안 제공
- **커뮤니티**: 이벤트 참가자들 간의 활발한 정보 공유 및 소통을 위한 게시판
- **마이페이지**: 구매 내역, 찜 목록, 가입 커뮤니티 등 사용자 활동 통합 관리

## 4. 시스템 아키텍처 (Architecture)
- **헥사고날 아키텍처 (Hexagonal Architecture)**
  - 외부 기술(DB, 웹 프레임워크)과 비즈니스 로직의 철저한 분리
  - 유연한 도메인 모델 확장성 확보
- **도메인 중심 설계**:
  - Event - Session - Ticket으로 이어지는 계층적 구조
  - 1:N 관계를 통한 유연한 세션 구성 가능

## 5. 주요 기술적 과제 및 해결 (Technical Challenges)

### 5-1. 세션 중심 이벤트 관리 (v6 아키텍처)
- **문제**: 이벤트 하나에 여러 시간대와 장소가 섞여 있는 경우 기존 단일 모델로는 한계가 있음.
- **해결**: 이벤트를 메타데이터로 두고, 실제 모집과 운영은 '세션(Session)' 단위로 관리. 최저가/날짜 등은 세션과 티켓 정보를 실시간 집계하여 제공.

### 5-2. 리치 컨텐츠 시스템 (Tiptap Editor)
- **문제**: 단순 텍스트만으로는 이벤트의 매력을 설명하기 부족함.
- **해결**: Tiptap Editor를 연동하여 이미지, 링크, 포맷팅이 포함된 HTML 컨텐츠 지원. XSS 방지를 위해 DOMPurify를 활용한 Sanitization 수행.

### 5-3. 온/오프라인 통합 및 주소 검색
- **문제**: 오프라인 주소의 정밀도 부족 및 온라인 링크 유출 우려.
- **해결**: 
  - **오프라인**: Kakao 우편번호 API 연동으로 시/도, 시/군/구 단위의 정밀 필터링 구현.
  - **온라인**: 구매자에게만 링크를 마스킹 해제(Masking)하여 보안 강화.
